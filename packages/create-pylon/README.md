# @lindorm/create-pylon

Scaffolder CLI for new [Pylon](https://github.com/lindorm-io/monorepo/tree/main/packages/pylon) applications.

## Usage

```bash
npm create @lindorm/pylon@latest my-app
```

Run without a name to be prompted:

```bash
npm create @lindorm/pylon@latest
```

The scaffolder prompts for features and drivers, writes the project, installs dependencies, and initialises git.

## What gets scaffolded

After answering prompts you get a project with:

- `src/index.ts` — entry file with `Symbol.metadata` polyfill and `pylon.start()`
- `src/logger/index.ts` — shared `Logger` instance
- `src/pylon/amphora.ts` — `Amphora` instance
- `src/pylon/config.ts` — typed config loaded from YAML + env via `@lindorm/config`, with a zod schema matching your selected drivers
- `src/pylon/pylon.ts` — the `Pylon` instance, wired with your selected features, workers, and sources
- `src/types/context.ts` — `ServerHandler`, `ServerHttpMiddleware`, `ServerSocketMiddleware`, and `ServerSocketHandler` aliases
- `src/handlers/` + `src/routes/v1/example.ts` — a working example HTTP handler (when HTTP is selected)
- `src/listeners/ping.ts` + `src/handlers/ping-handler.ts` — a working example Socket.IO listener (when Socket.IO is selected)
- `src/routes/webhooks/` — CRUD routes for `WebhookSubscription` (when webhooks are selected)
- `src/workers/` — built-in worker files (`createAmphoraEntityWorker` etc.) + an `alive.ts` example
- `src/proteus/source.ts` + `src/proteus/entities/SampleEntity.ts` — generated by `proteus init` + `proteus generate entity` after install (when a proteus driver is selected)
- `src/iris/source.ts` + `src/iris/messages/SampleMessage.ts` + sample publisher + subscriber — generated by `iris init` + `iris generate message` after install (when an iris driver is selected)
- `docker-compose.yml` — containers for the selected drivers (when any driver needs one)
- `config/default.yml` — default config values
- `.env` — environment variables with driver-specific defaults
- `package.json`, `tsconfig.json`, `tsconfig.build.json`, `jest.config.ts`, `.gitignore`

## Prompt flow

```
? Project name: my-app
? Select features: (HTTP routes / Socket.IO listeners)
? Persistence driver (Proteus): (none / memory / mongo / mysql / postgres / redis / sqlite)
? Message bus driver (Iris): (none / kafka / nats / rabbit / redis)
? Webhooks? (y/N)              (only when both proteus and iris are selected)
? Audit logging? (y/N)         (only when both proteus and iris are selected)
? Workers: (Amphora key refresh / Amphora entity sync / Expiry cleanup / Kryptos key rotation)
```

The proteus-dependent workers only appear when a proteus driver is selected. Queue and rooms are auto-enabled when iris or socket is selected respectively.

## Programmatic use

```typescript
import { scaffold, runPrompts } from "@lindorm/create-pylon";

const answers = await runPrompts();
await scaffold(answers);
```

See `src/types.ts` for the `Answers` shape.
